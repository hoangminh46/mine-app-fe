# ❌ Những Sai Lầm Thường Gặp Khi Viết CV Developer

> Checklist các lỗi phổ biến mà lập trình viên hay mắc phải,
> kèm cách sửa cụ thể cho từng lỗi.

---

## 1. Sai Lầm Về Nội Dung

### ❌ Liệt kê nhiệm vụ thay vì thành tựu
```
SAI:  "Responsible for maintaining backend APIs"
      "Worked on frontend features using React"

ĐÚNG: "Optimized 15 backend APIs, reducing avg response time by 40%"
      "Built 8 customer-facing features in React, increasing user engagement by 25%"
```
> **Quy tắc:** Mỗi bullet point phải trả lời: "So what? Kết quả là gì?"

### ❌ Professional Summary quá generic
```
SAI:  "Seeking a challenging role where I can utilize my skills and grow professionally"

ĐÚNG: "Frontend Engineer with 3 years building React/TypeScript apps. 
       Reduced bundle size by 45% at XYZ Corp. Passionate about performance and DX."
```

### ❌ Liệt kê quá nhiều skills "rác"
```
SAI:  HTML, CSS, JavaScript, TypeScript, React, Vue, Angular, Svelte, 
      Node.js, Python, Java, Go, Rust, C++, SQL, MongoDB, Redis, 
      Firebase, AWS, Azure, GCP, Docker, Kubernetes...

ĐÚNG: Languages: JavaScript, TypeScript, Python
      Frameworks: React.js, Next.js, NestJS
      Databases: PostgreSQL, Redis
      Cloud/DevOps: AWS (EC2, S3, Lambda), Docker, GitHub Actions
```
> **Quy tắc:** Chỉ liệt kê 10-12 skills bạn tự tin phỏng vấn được.

### ❌ Thiếu con số / metrics
```
SAI:  "Improved application performance"
ĐÚNG: "Improved application performance by reducing LCP from 3.2s to 1.4s"

SAI:  "Worked with a large team"  
ĐÚNG: "Collaborated in a cross-functional team of 12 (5 devs, 3 QA, 2 PM, 2 designers)"
```

### ❌ Dùng từ yếu / passive
```
SAI:  "Was involved in...", "Helped with...", "Familiar with...", "Worked on..."
ĐÚNG: "Engineered...", "Spearheaded...", "Optimized...", "Deployed...", "Architected..."
```

## 2. Sai Lầm Về Format

### ❌ CV quá dài
- 3 năm KN → **PHẢI giữ 1 trang**
- Mọi thông tin không liên quan → CẮT

### ❌ Dùng template Canva nhiều màu sắc
- ATS không đọc được → CV bị loại trước khi recruiter thấy
- **Giải pháp:** Dùng Overleaf, SWEResume, hoặc Google Docs

### ❌ Layout 2 cột
- ATS đọc từ trái sang phải, top to bottom
- 2 cột → thông tin bị xáo trộn
- **Giải pháp:** Luôn dùng 1 cột

### ❌ Contact info trong header/footer
- Nhiều ATS không parse được header/footer
- **Giải pháp:** Đặt contact info trong body text, phần đầu trang

### ❌ Dùng icons, emojis, progress bars
```
SAI:  📧 email@gmail.com  |  ⭐⭐⭐⭐⭐ JavaScript
ĐÚNG: Email: email@gmail.com  |  JavaScript (liệt kê trong Technical Skills)
```

### ❌ File format sai
- Không gửi .doc, .pages, hoặc image PDF
- **Luôn gửi:** PDF text-based (text selectable) hoặc .docx

## 3. Sai Lầm Về Chiến Lược

### ❌ Gửi 1 CV cho tất cả công ty
- Mỗi JD có keywords khác nhau
- ATS chấm điểm dựa trên keyword match
- **Giải pháp:** Tailor CV cho từng vị trí (ít nhất Summary + Skills)

### ❌ Không có GitHub/Portfolio link
- Với dev, đây là "bằng chứng" quan trọng nhất
- CV nói bạn biết React → GitHub CHỨNG MINH bạn biết React
- **Giải pháp:** Luôn có GitHub link trong header

### ❌ LinkedIn không khớp với CV
- Recruiter LUÔN cross-check
- Nếu timeline hoặc skills không khớp → Red flag

### ❌ Không test ATS trước khi nộp
- Copy CV → paste vào Notepad → đọc được = OK
- Dùng Jobscan để check score

### ❌ Chỉ nộp qua hệ thống, không networking
- Tỷ lệ response khi nộp "cold" qua ATS: ~5-10%
- Tỷ lệ khi có referral hoặc gửi email trực tiếp: ~30-50%
- **Giải pháp:** Tìm recruiter/hiring manager trên LinkedIn, gửi message kèm CV

## 4. Sai Lầm Đặc Thù Thị Trường VN

### ❌ CV tiếng Việt cho công ty quốc tế
- Outsourcing, MNC → PHẢI tiếng Anh

### ❌ Tự đánh giá skills bằng % hoặc stars
```
SAI:  Java ████████░░ 80%
ĐÚNG: Java (liệt kê trong Technical Skills, không rating)
```
> Nhà tuyển dụng VN cũng ngày càng coi đây là unprofessional.

### ❌ Ghi quá nhiều thông tin cá nhân
- Không cần: CMND, ngày sinh, tình trạng hôn nhân, tôn giáo
- Chỉ cần: Tên, SĐT, Email, City, GitHub, LinkedIn

### ❌ Nhảy việc liên tục (<1 năm/công ty)
- Thị trường VN đánh giá loyalty khá cao
- Nếu đã nhảy nhiều: focus vào achievements, không highlight dates

---

## 5. Quick Fix Checklist

Trước khi nộp CV, kiểm tra:

- [ ] Mỗi bullet point có ACTION VERB + METRIC không?
- [ ] CV có vừa 1 trang không?
- [ ] Layout 1 cột, không có graphics/icons?
- [ ] Contact info trong body (không phải header/footer)?
- [ ] GitHub link có trong header không?
- [ ] Đã tailor keywords theo JD cụ thể?
- [ ] Đã test ATS (paste vào Notepad)?
- [ ] Đã proofread grammar & spelling?
- [ ] File PDF < 1MB, text selectable?
- [ ] LinkedIn profile khớp với CV?

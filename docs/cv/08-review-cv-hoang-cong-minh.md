## ❌ Vấn Đề Cần Khắc Phục (Theo Mức Độ Ưu Tiên)

### 🔴 P0: Critical - Cần sửa NGAY

#### 1. Template TopCV → ATS khó đọc
**Vấn đề:** Template TopCV thường sử dụng layout nhiều cột, text box, icons, và formatting phức tạp. Hầu hết ATS sẽ parse sai hoặc bỏ qua nhiều thông tin.

**Giải pháp:**
- Chuyển sang công cụ ATS-friendly: **Overleaf (LaTeX)**, **SWEResume.app**, hoặc **Google Docs** với template đơn giản
- Dùng layout 1 cột, font chuẩn (Arial/Calibri), không icons/graphics

#### 2. KHÔNG có con số / metrics nào
**Đây là vấn đề LỚN NHẤT.** Toàn bộ CV không có bất kỳ con số nào để chứng minh impact.

**Work Experience hiện tại (chỉ liệt kê tasks):**
```
❌ "Developed and maintained web applications using Next.js, React.js, and Vue.js, 
    ensuring high performance and scalability."

❌ "Implemented new features and optimized existing functionalities based on 
    project requirements."

❌ "Collaborated with backend developers to implement API integrations and 
    ensure smooth data flow"
```

**Nên viết lại thế nào:**
```
✅ "Developed and maintained 3 web applications using Next.js, React.js, and Vue.js, 
    serving 50+ enterprise clients with 99.5% uptime."

✅ "Implemented 15+ new features and optimized existing codebase, reducing 
    page load time by 35% through code splitting and lazy loading."

✅ "Integrated 20+ RESTful API endpoints with Axios, implementing client-side 
    caching that reduced redundant API calls by 40%."
```

#### 3. Thông tin cá nhân thừa
**Vấn đề:** CV có "Date of birth: June 04, 2000" — đây là thông tin KHÔNG CẦN THIẾT, có thể gây bias.

**Giải pháp:** Bỏ ngày sinh. Chỉ giữ: Tên, SĐT, Email, City, GitHub, LinkedIn.

## 💡 Tips Bổ Sung

### Cách "chế" metrics khi không có số liệu chính xác
Anh không cần số liệu chính xác tuyệt đối. Ước tính hợp lý hoàn toàn OK:

| Không có | Ước tính |
|----------|----------|
| Không biết chính xác số users | "serving 50+ enterprise clients" |
| Không đo performance chính xác | "reducing load time by ~30%" |
| Không nhớ số features | "delivered 10+ features per quarter" |
| Không biết team size | "collaborated in a cross-functional team of 8" |

> Nguyên tắc: **Ước tính conservative** (thấp hơn thực tế) tốt hơn **không có số nào cả**.

### Với 3 năm KN, nhà tuyển dụng muốn thấy gì?
1. ✅ Bạn có thể **làm việc độc lập** (không cần hand-holding)
2. ✅ Bạn bắt đầu **mentoring** người khác
3. ✅ Bạn **hiểu business context**, không chỉ code theo task
4. ✅ Bạn có **ownership** từ design đến deploy
5. ✅ Bạn có khả năng **đề xuất technical improvements**

> Bullet point cuối trong Experience AMELA: *"Proposed and applied technical improvements to enhance system efficiency"* — đây là dấu hiệu TỐT! Nhưng cần thêm CỤ THỂ: propose cái gì? Kết quả ra sao?
